# Approved Text - Progression Scale

## Core Pacing Idea

* The game starts painfully slow.
* Small numbers should matter early.
* Later progression should accelerate strongly through unlocked words, paths, and sentence combos.
* The player should feel that smart choices can change progression speed.
* Progress should move through orders of magnitude: 0.01 -> 0.1 -> 1 -> 10 -> 100 -> 1K -> 10K -> 100K -> 1M -> 1B -> 1T+.

## Meaning Scale

* 0.01 to 2 Meaning: slow manual World stamping.
* 2 Meaning: first path choice.
* 10 Meaning: next word in chosen path.
* 100 Meaning: first major structural unlock, likely second sentence slot.
* 1,000 Meaning: first modifier/early combo layer.
* 10,000 Meaning: stronger path synergy.
* 100,000 Meaning: automation or major acceleration tool.
* 1,000,000 Meaning: first prestige/New Edition territory.
* 1B+ Meaning: deep systems, grammar/reality editing later.

## Progression Philosophy

* Every new number range should unlock a new kind of tool.
* Do not add tools randomly.
* Each tool should change how Meaning is generated.
* Early game should be slow, but not boring.
* Later game should reward the grind with explosive progression.
* Explosive progression must be controlled by limits: sentence slots, path restrictions, word types, costs, instability later, and prestige.

## Player Speed Bands

* Low path: passive or weak choices, slower progress.
* Average path: normal choices, expected pacing.
* High path: smart combos, active play, faster progress.
* Random path: unreliable, sometimes huge bursts.

## Current Immediate Balance

* World starts at +0.01 Meaning/tap.
* First path choice currently appears at 2 Meaning.
* First choice should remain early enough to avoid boredom, but still feel earned.
* Farm/Manual path should improve active tapping.
* Water/Flow path should improve passive Meaning/sec.
* Rain currently provides +0.006 Meaning/sec to smooth the Water 10 -> 25 segment.
* Dream/Chance path should appear later and focus on rare high-value events.

## Current Safety Constants

* Meaning Bloom grants max(current Meaning * 10%, current tap gain * 10).
* Final upgrade costs use max(discounted cost, base cost * 25%, 0.1 Meaning).
* Normal event delay has a hard minimum of 90 seconds after frequency bonuses.

## Current Design Rule

* Do not build new mechanics just because they are cool.
* Build only the next tool needed for the next Meaning range.
* Balance by "time to next interesting decision," not by raw numbers alone.
